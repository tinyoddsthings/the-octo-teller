# 創角 / 開新戰役 完整流程

> 本檔整合「開始 D&D」流程 + 創角步驟 1-12 + 角色卡欄位計算規則 + 開新戰役指令。
> 日常遊玩不需載入，創角/開戰役時才讀。

---

## 開始 D&D 時的流程
1. 詢問玩家：「要繼續哪個戰役，還是開新戰役？」
2. 列出 `memory/dnd/campaigns/` 底下現有的戰役資料夾
3. 選擇後讀取該戰役的 `campaign.md`
4. 開新戰役時：
   - 列出可用劇本（dm20 library 已載入的模組），讓玩家選
   - 詢問戰役名稱。不命名則預設用「劇本名稱_YYYYMMDD」命名資料夾（如 `lost-mine-of-phandelver_20260228`）
   - 選擇難度，列出四種選項讓玩家選（詳見 `reference/dm_settings/difficulty_styles.md`）：
     1. 菜雞（Beginner）— 新手友善，微調幫助，低滅團風險
     2. 傳統（Traditional）— 純 RAW，不微調，骰子決定一切
     3. 效果（Dramatic）— 戲劇張力最大化，雙向微調
     4. 高手（Expert）— 高挑戰，策略導向，微調增難
   - 選擇敘事風格，列出六種選項讓玩家選（詳見 `reference/dm_settings/narrative_styles.md`）：
     1. 簡潔（Concise）— 短句重點，快節奏
     2. 小說（Literary）— 感官細節，文學腔
     3. 電影（Cinematic）— 鏡頭感，慢動作特寫
     4. 搞笑（Comedic）— 吐槽輕鬆，偶爾破牆
     5. 黑暗（Gritty）— 殘酷寫實，強調代價
     6. 言情（Romance）— 誇張情感，曖昧描寫
   - 確認玩家人數與隊伍組成：
     1. 詢問「這次有幾位玩家參與？」
     2. 查詢劇本建議人數（從 dm20 library / PDF 取得）並告知玩家
     3. 若實際人數 < 建議人數，先問「要增加 NPC 隊友嗎？」（說明利弊）
     4. 若要補 → 再問「要補幾位？」（建議補到劇本建議人數下限）。一次只問一個問題，不要同時問要不要補和補幾位
     5. 確認遊玩模式（1 人 = 單人，2 人以上 = 多人）。不需另外問玩家名字，用角色名識別即可
   - 規則版本預設 2024 PHB，不特別詢問（玩家主動提出才切換）
   - 依人數逐一進入創角流程（先創玩家角色，再創 NPC 隊友）。角色名在創角第 1 步詢問
   - 建立資料夾結構（campaign.md 格式見 `reference/campaign_template.md`）

## 創角流程

使用 `reference/角色卡_空白.md` 作為模板，一次問一個問題，順序如下。
**所有選項都要加編號，讓玩家可以直接打數字選擇。**
選項表格見 `reference/creation/*.md` 系列檔案。
**⚠️ 呈現選項時，必須完整照抄 `reference/creation/*.md` 的內容，禁止濃縮、改寫或省略。**

1. **角色名稱** — 取名或讓玩家命名
2. **種族** — 列出總表讓玩家選（打數字）。有子選項的種族用複合編號（如 2-1 = 龍裔黑龍先祖、10-1 = 提夫林深淵傳承），可一步選完種族+細項。沒有子選項的直接跳下一步。種族特性標示解鎖等級（如 `Lv.3 解鎖`、`Lv.5 解鎖`）。注意：2024 PHB 無傳統子種，改為血統（Lineage）/ 傳承（Legacy）/ 先祖（Ancestry）選項
   - **種族施法能力**（有種族法術的種族才問）— 選完種族+子選項後，**必定馬上問**施法能力，不可跳過或延後。印出以下固定格式表格讓玩家選：
     ```
     ┌─────┬─────────────┬───────────────────────────────────────────────────────────────────┐
     │  #  │  施法能力   │                          適合搭配的職業                           │
     ├─────┼─────────────┼───────────────────────────────────────────────────────────────────┤
     │ 1   │ 智力（INT） │ 知識路線：法師（Wizard）                                           │
     ├─────┼─────────────┼───────────────────────────────────────────────────────────────────┤
     │ 2   │ 感知（WIS） │ 直覺洞察路線：牧師（Cleric）/ 德魯伊（Druid）/ 遊俠（Ranger）/    │
     │     │             │ 武僧（Monk）                                                      │
     ├─────┼─────────────┼───────────────────────────────────────────────────────────────────┤
     │ 3   │ 魅力（CHA） │ 社交路線：魔導士（Warlock）/ 術士（Sorcerer）/ 吟遊詩人（Bard）/   │
     │     │             │ 聖騎士（Paladin）                                                 │
     └─────┴─────────────┴───────────────────────────────────────────────────────────────────┘
     ```
   - 選項表見 `reference/creation/species.md`
3. **職業** — 列出可選職業（含核心特色說明），選後填入職業特性、熟練項目（護甲/武器/工具）、豁免熟練
   - 選項表見 `reference/creation/classes.md`
4. **背景** — 列出可選背景，開頭提示「不確定名詞可以先選，選完會給你看詳細說明」。
   玩家選編號後，印出起源專長全文機制 + 技能熟練用途說明 + 工具熟練用途說明，
   問「確認還是想看其他的？」。確認後填入技能/工具熟練與起源專長。
   能力值加成留到第 6 步處理。
   **魔法新手（Magic Initiate）標示規則：** 預覽時列出機制全文，但戲法、法術、施法能力的具體選擇標示「稍後選擇技能時再決定」，不在此步驟做。
   **裝備呈現規則：** 所有背景預覽裝備時，不寫「裝備 A」，寫「裝備：[物品清單]」，下方加註「（也可以不拿裝備，改拿 50 GP，在選擇職業裝備時讓你決定）」。
   - 選項表見 `reference/creation/backgrounds.md`
5. **語言** — 通用語（Common）自動獲得 + 從標準語言表選 2 種。列出標準語言表（含使用者/來源），讓玩家選或擲骰（1d12）
   - 語言表見 `reference/creation/options.md`
6. **能力值** — 三種方式全部印出（標準陣列含該職業建議配置、擲骰、點數購買），讓玩家選方式後再分配。修正值說明不在此步驟講解，等分配完畢算衍生數值時再解釋
   - **點數購買互動流程：** 玩家選擇點數購買後，不給建議配置，直接進入互動分配（格式見 `workflows/presentation_templates.md`）
   - 選項表見 `reference/creation/abilities.md`
7. **技能選擇** — 根據職業+背景可選清單，列出每個技能的用途說明。表頭用「相關能力」，能力欄用「中文（英文縮寫）」格式（如「感知（WIS）」）
   - 技能表見 `reference/creation/options.md`
8. **起源專長**（2024 PHB）— 列出背景提供的專長選項，附利弊
   - 專長表見 `reference/creation/options.md`
9. **法術/戲法**（施法職業）— 依法術表格式呈現可選法術（含 C/R/M 標記），附實戰場景舉例
   - 法術清單見 `reference/creation/options.md`；格式規範見 `workflows/presentation_templates.md`
9a. **魔寵選擇**（僅限有魔寵的職業/祈喚，如契約之鎖）—
    - 列出所有可選魔寵（完整編號），讓玩家選
    - **⚠️ 呈現選項時，必須完整印出 `reference/class_progression/{職業}.md` 魔寵段落的全部內容：敘述 + 完整數值表格，不可省略任何欄位**
    - 玩家選定後，將魔寵完整資訊登記到角色卡「魔寵」區塊（見下方格式）
10. **裝備** — 職業起始裝備或購買，列出選項
11. **角色塑造**（外觀 + 性格 + 背景故事）—
    - 11a. 問「你有已經設定好的 D&D 角色人設嗎？」
      - **有** → 再問「要自己描述，還是用引導問題幫你整理？」自由描述則玩家寫、DM 整理；引導問題則用問卷框架但問的是角色
      - **沒有** → 問「想不想填一份關於你自己的小問卷？我會把你的真實個性轉換成 D&D 角色」。同意則進入問卷，不要則跳過（遊戲中逐步發展）
    - 11b. **現實→D&D 問卷**（DM 根據主題清單即時生成，自述回答可跳過已涵蓋主題）：
      - DM 根據 `reference/character_quiz.md` 的主題清單即時生成題目和選項，每次創角題目不同
      - 每題 6 個選項：1-4 為 DM 生成的預設答案，5 = 跳過，6 = 自己描述
      - 自述跳題：玩家選「自己描述」時，DM 分析回答涵蓋的主題，已涵蓋的後續不再出題
      - 每題回答後顯示「`（了解進度：X%）`」= 已收集主題 / 主題清單總數，100% 時主動結束
      - 跳過的題目 DM 根據已答題目 + 角色卡既有資訊（種族/職業/背景）自動生成，不告知玩家影響
      - **一次問一題**，等玩家回答後再印下一題（與創角其他步驟一致）
      - 問卷不出現 D&D 術語，全用日常情境
      - **主題清單、出題規則及 DM 轉換映射表見 `reference/character_quiz.md`**
    - 11c. DM 將回答轉換為 D&D 設定（轉換映射見 `reference/character_quiz.md`），一次呈現結果，玩家逐項確認或調整
    - 11d. 確認後寫入角色卡
    - **性格/理念/關係/缺點** — 問卷生成後標註「初始設定，遊戲中會演變」；跳過問卷則創角不填、Session 結束後歸納
12. **語音設定** —
    - **玩家角色**：創角時留空，遊戲進行中從玩家實際發言風格歸納，Session 結束時自動填入/更新
    - **NPC 角色**：DM 自動生成語音設定（基礎語氣 + 口頭禪 + 六種風格台詞），參照 `reference/dm_settings/narrative_styles.md`

每一步完成後立即更新角色卡對應區塊，不要等全部選完才寫。

## 角色卡欄位計算規則

填寫角色卡時，所有衍生數值必須展示計算過程：

- **AC** = 護甲基礎 + DEX修正 + 盾牌 + 其他（寫在備註）
- **法術攻擊** = 熟練加值 + 施法能力修正
- **法術 DC** = 8 + 熟練加值 + 施法能力修正
- **技能加值** = 能力修正 + 熟練加值（如熟練）
- **HP** = 職業生命骰 + CON修正×等級 + 種族加成
- **被動感知** = 10 + 感知修正 + 熟練加值（如熟練感知）

## 開新戰役指令（快速版）

玩家說「開新戰役」時，DM 照以下流程逐步執行（一次一題）：

1. 問戰役模組名稱（或自訂冒險）
2. 如有新 PDF → 用 dm20 載入
3. 問玩家模式（單人/多人）
4. 問難度（菜雞/傳統/效果/高手）
5. 問敘事風格（簡潔/小說/電影/搞笑/黑暗/言情）
6. 建立 campaign 目錄結構（campaign.md + sessions/ + 角色卡）
7. 進入創角流程（步驟 1-12，照上方「創角流程」走）

規則：
- 一次只問一個問題，等回答再問下一題
- 不進 plan mode，不一次問多個問題
- 開發者偵錯模式：玩家指定時啟用，遇不確定規則立即暫停查 PDF
